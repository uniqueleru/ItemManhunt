마인크래프트 아이템 맨헌트 플러그인 기획서

[유의사항]

이 플러그인은 향후 기능 추가의 용이성과 유지보수의 편의성을 위해 다중 파일 구조로 작성되어야 합니다.
코드를 작성할때는 엣지 케이스에 주의해야하며 버그없이 완벽하게 동작하는 코드를 작성해야합니다.
메세지, 타이틀, 액션바와 같이 플레이어에게 표시되는 모든 메세지는 lang.yml 에서 설정이 가능해야합니다.
플러그인의 모든 세부 설정은 config.yml 에서 수정이 가능해야합니다.

[게임의 규칙]

게임 시작전, 술래 한 명을 지정합니다.
게임이 시작되면 술래는 도망자가 되고, 서버의 나머지 플레이어들은 추격자가 됩니다.
도망자는 게임이 시작될때 랜덤하게 정해지는 '목표 아이템' 을 한번도 죽지 않고 획득하면 승리하며, 추격자는 도망자가 이 목표 아이템을 얻기 전에 처치해야합니다.
게임이 시작되면 플레이어 전원에게 도망자의 '목표 아이템' 이 타이틀로 공지되고, 도망자에게 먼저 시작할 시간이 일정 시간 주어집니다. 이때 추격자들은 움직일 수 없습니다.
추격자들의 추격이 시작되면 플레이어 전원에게 해당 사실이 공지되고 추격자들에겐 도망자의 위치를 추적할 수 있는 나침반이 주어집니다.
도망자가 '목표 아이템'을 획득하면 즉시 도망자의 승리로 게임이 종료되고, 플레이어 전원에게 타이틀로 해당 사실이 공지됩니다.
도망자가 '목표 아이템' 획득 전에 사망하면 즉시 추격자들의 승리로 게임이 종료되고, 플레이어 전원에게 타이틀로 해당 사실이 공지됩니다.

[목표 아이템 설정 GUI]

/imh config 명령어로 목표 아이템 설정 GUI 를 열 수 있습니다.
이 GUI 는 다중 페이지로 구성되어 있어 다음 페이지와 이전 페이지 버튼이 있으며, 마인크래프트에 존재하는 모든 아이템들이 나열되어 있습니다.
로어로 해당 아이템의 활성화/비활성화 여부가 표시되어 있으며, 클릭을 통해 토글 할 수 있습니다. 이 GUI 에서 활성화 되어 있는 아이템은 도망자의 '목표 아이템' 으로 설정 될 수 있습니다.
GUI 관련 코드 작성시 인벤토리 제목으로 플레이어가 GUI 를 연 상태인지 확인하는 방식으로 로직을 구성해야합니다.


[명령어]

/imh - 플러그인의 명령어 도움말을 봅니다.
/imh setrunner [닉네임] - 다음 게임의 도망자를 지정합니다.
/imh start - 게임을 시작합니다.
/imh stop - 게임을 강제 종료합니다.
/imh reload - 플러그인의 설정 파일을(config.yml, lang.yml) 리로드합니다.
/imh config - '목표 아이템' 으로 지정될 수 있는 아이템 목록을 설정하는 GUI 창을 엽니다.
